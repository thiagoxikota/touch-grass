## Design Context

### Users
Primary audience is mixed: potential Timeouts users plus design/engineering peers evaluating quality and rigor.  
Primary jobs to be done are: (1) convert visitors to waitlist/download intent, and (2) communicate brand credibility plus practical DS adoption paths (npm install, docs exploration, system understanding).

### Brand Personality
Three-word personality: **bold, young, productivity**.  
Tone should feel opinionated and energetic, but still technically trustworthy.

### Aesthetic Direction
Dark-first interface with optional light support.  
Direction should blend the conversion clarity of strong app landing pages with the trust signals and structure of top design-system documentation sites.  
Visual creativity is required (more compelling visual assets/composition), while staying brutalist-core (zero motion, strong contrast, hard edges).

### Design Principles
1. **Conversion + credibility together**: every major section should either drive waitlist intent or prove system quality (ideally both).
2. **Creative but disciplined visuals**: use imagery/assets and asymmetric composition for personality, without breaking token, contrast, and layout rules.
3. **Accessibility is non-negotiable**: WCAG AA minimum across all text/background pairings; no low-contrast accents.
4. **Dark-first brutalist consistency**: preserve zero-radius, zero-motion, token-only behavior as foundational constraints.
5. **Actionable documentation UX**: make install paths, component proof, and cross-links obvious within one scroll.
